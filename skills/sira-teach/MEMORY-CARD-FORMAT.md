# Memory Card Format

Use this format for old-friends cards. Cards can live in one `memory/cards.md` file or as separate files when a deck grows.

```md
### Card: {short name}
- **ID**: {stable id, e.g. geography-continents-asia}
- **Prompt**: {what the parent asks, shows, or invites the child to do}
- **Answer / cue**: {acceptable answer, gesture, pointing action, performance cue, or observation}
- **Type**: {fact | map image | place to map | gesture | chant | story anchor | procedure | marvel | route | local clue | other}
- **Why keep**: {why this is worth future re-meeting}
- **Chosen by**: {child | parent | Sira | shared}
- **Source**: {lesson, practice, learning record, resource, or parent report}
- **Due**: {YYYY-MM-DD}
- **Interval**: {plain interval, e.g. 3 days, 1 week, 1 month}
- **Ease**: {low | normal | high}
- **Status**: {active | paused | retired}
- **Last result**: {Again | Hard | Good | Easy | not yet reviewed}
```

## Rules

- Keep prompts short enough for a parent to use without studying.
- Give every card a stable `ID` before embedding it in HTML or logging reviews.
- Let cards be embodied: pointing, drawing, singing, acting, finding, and explaining all count.
- Prefer cards the child chose, loved, used, or found strange. If the card is adult-chosen, make the reason visible in `Why keep`.
- Use `Again`, `Hard`, `Good`, and `Easy` only as scheduling signals, not as moral judgments.
- Retire cards that have become too easy or irrelevant.
- If reviews pile up, reschedule gently instead of creating a backlog.
- For review rhythm and card-worthiness, read `MEMORY-BOX-GUIDE.md`.
- For HTML review logs, read `OLD-FRIENDS-HTML-STANDARD.md`.
