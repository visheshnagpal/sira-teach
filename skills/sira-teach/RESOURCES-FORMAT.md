# RESOURCES.md Format

`RESOURCES.md` is the family's curated set of trusted, age-fit sources for a subject. Knowledge for lessons is drawn from here, not from parametric guesses. Wisdom comes from the real-world places, practices, and people listed here.

> The matching domain in [USE-CASE-CATALOG.md](./USE-CASE-CATALOG.md) is your starting menu — it names specific books, apps, and sites per topic. Treat those as *leads to vet*, not endorsements: confirm each is right for this child's age and stage before it earns a place here.

## Structure

```md
# {Subject} Resources — for {child or family}

## Knowledge (age-appropriate)

- [Picture book: _The Tiny Seed_ — Eric Carle](https://example.com)
  Beautiful, simple germination story. Use for: the first lesson on how plants begin. Read together.
- [Video: "How a seed grows" — {trusted kids' channel}](https://example.com) (4 min)
  Time-lapse footage. Use for: the wonder hook. Watch together, pause to wonder aloud.

## Wisdom (real-world practice — supervised)

- The garden / a window box at home
  Use for: planting and watching a real seed. The tightest feedback loop there is.
- Saturday children's session at {local botanical garden / library}
  Use for: doing it alongside other kids. Parent-accompanied.

## Family constraints and preferences

- {Screen-time limits, budget, religious/cultural fit, library access, local places, books already owned, disliked formats}

## Substitutes

- {If the preferred source is unavailable, what can be used instead?}
```

## Rules

- **Age-appropriate AND high-trust.** A source can be correct and still be wrong for this child — too advanced, too dense, too scary. Both bars must clear. Note the format (picture book, short video) and whether it's read/watched *together* or independently.
- **Annotate every entry.** One line: what it covers and when to reach for it. A bare link is useless in three months.
- **Group by Knowledge / Wisdom.** Wisdom for a child is real-world, supervised practice — family, show-and-tell, museums, nature, a vetted local class. **Never** a forum or subreddit.
- **Surface gaps explicitly.** If no good age-appropriate resource exists for an area the mission needs, write a `## Gaps` section. This drives future search — and may mean *you* must create the resource as a reference doc.
- **Prune ruthlessly.** Five sharp, well-pitched sources beat thirty mediocre ones.
- **Record the operator's wishes.** If the parent has ruled out a kind of community or a screen-time format, note it here so future sessions don't keep proposing it.
